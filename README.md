## committing convention & naming convention

### naming convention

- 문서의 제목은 50자를 넘기지 않도록 유의한다.
- 제목만 봐도 무슨 내용인지 알 수 있도록 적는다.

### committing convention

- **수정시**

1. fix : 버그 수정
2. refactor : 코드 리팩토링
3. docs : 문서 수정

- **일반 업로드시**

1. [날짜 (년도 월 일)] : 업로드 내용 간략 기술 (문서 제목)
